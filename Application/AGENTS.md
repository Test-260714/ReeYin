# Application 领域规则

本文件继承根级 [AGENTS.md](../AGENTS.md)。风险和快速通道以 [CONTRIBUTING.md](../CONTRIBUTING.md#3-风险分类与快速通道) 为准；本文件只增加 Application 特有约束。

## 适用范围与职责

适用于 `Application/` 下的配置、初始化、登录/权限、报警、状态、Recipe、项目管理、节点编排、共享应用服务和 WPF 辅助项目。修改前从 [变更影响地图](../docs/development/change-impact-map.md) 定位消费者，并从 [构建与测试地图](../docs/development/build-and-test-map.md) 选择验证入口。

## 修改前调查

- 定位 Prism 注册/导航、服务接口、全局事件、后台任务、配置与持久化、权限检查、报警传播和退出释放路径。
- 检查 `Shell`、UI、Tools、CustomizedDemand 及动态模块消费者；项目引用不是动态消费者的完整证明。
- 区分共享应用契约与单页面内部行为，确认线程、取消、超时、重复初始化和资源所有权。

## 升级条件

- 公共服务、配置、Recipe、项目/方案、导航身份或跨模块事件变化至少 R2。
- 报警严重度/确认/清除/抑制语义、权限判定、安全状态或发布配置变化至少 R3。
- 涉及真实设备或生产数据时遵守独立授权；不可逆受控操作为 R4，AI 不得执行。

## 最低验证

- 构建修改项目并运行直接测试；公共契约继续构建可定位的 Shell、UI、Tools 或客户模块消费者。
- UI 覆盖 Binding、命令、导航、关闭/取消和线程；报警/权限覆盖拒绝、失败、恢复和审计语义。
- 交付说明配置、数据、用户行为、权限、报警和启动/退出兼容性，以及未验证项和回滚。
