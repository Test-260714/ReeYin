# RecipeManagerView 重新设计总结

## 设计目标
参考HardwareConfigView的左侧导航设计，重新设计RecipeManagerView页面，简化界面结构，提高用户体验。

## 主要改进

### 1. 布局结构优化
**原设计：** 左侧白色列表 + 右侧详情区 + 节点分组展示
**新设计：** 左侧深色侧边栏导航 + 右侧配方详情区

```
┌─────────────────────────────────────────┐
│  工具栏（新建、复制、删除等操作）        │
├──────────────┬──────────────────────────┤
│              │                          │
│  深色侧边栏  │   配方详情区             │
│  配方列表    │   - 基本信息             │
│  当前配方    │   - 审计信息             │
│              │   - 描述                 │
├──────────────┴──────────────────────────┤
│  状态栏（保存状态、存储位置）            │
└─────────────────────────────────────────┘
```

### 2. 左侧侧边栏设计
- **背景色：** 深色主题 (#1E293B)
- **列表项样式：** 
  - 正常状态：透明背景
  - 悬停状态：浅灰背景 (#334155)
  - 选中状态：蓝色背景 (#2563EB)
- **文本颜色：** 浅灰色 (#CBD5E1)
- **显示内容：**
  - 配方名称
  - 配方分类
  - 当前配方指示区

### 3. 右侧详情区设计
- **空状态：** 显示提示信息和图标
- **详情区包含：**
  - 配方标题和摘要
  - 基本信息（名称、分类、版本、作者）
  - 审计信息（创建时间、更新时间、修改人、启用状态）
  - 配方描述

### 4. 移除的内容
- ✂️ 节点分组展示（DataGrid）
- ✂️ 参数编辑表格
- ✂️ 节点分组标题和列表

**说明：** 参数编辑功能已移至RecipeParamEditView对话框中，通过"编辑参数"按钮打开。

### 5. 颜色方案
| 用途 | 颜色 | 十六进制 |
|------|------|---------|
| 侧边栏背景 | 深灰蓝 | #1E293B |
| 侧边栏悬停 | 浅灰蓝 | #334155 |
| 侧边栏选中 | 蓝色 | #2563EB |
| 侧边栏文本 | 浅灰 | #CBD5E1 |
| 主面板背景 | 浅灰 | #F2F5F9 |
| 卡片背景 | 白色 | #FFFFFF |
| 主文本 | 深灰 | #1A2236 |
| 副文本 | 中灰 | #6B7A99 |

### 6. 交互流程
1. **查看配方列表** → 左侧侧边栏显示所有配方
2. **选择配方** → 右侧显示配方详情
3. **编辑基本信息** → 直接在右侧表单编辑
4. **编辑参数** → 点击"编辑参数"按钮打开RecipeParamEditView对话框
5. **保存** → 点击"保存"按钮保存所有更改

## 用户体验改进

### 优点
✅ **简洁清晰** - 移除复杂的节点分组展示，界面更清爽
✅ **导航直观** - 深色侧边栏提供清晰的视觉层级
✅ **操作高效** - 参数编辑通过专用对话框，避免主界面混乱
✅ **响应式设计** - 左侧固定宽度，右侧自适应
✅ **一致性** - 参考HardwareConfigView设计，保持应用风格统一

### 功能完整性
- ✅ 配方列表浏览
- ✅ 配方基本信息编辑
- ✅ 配方参数编辑（通过对话框）
- ✅ 配方创建、复制、删除
- ✅ 配方激活（设为当前）
- ✅ 配方保存和加载
