# Shell 领域规则

本文件继承根级 [AGENTS.md](../AGENTS.md)，只增加组合根和运行时装配约束。风险定义见 [CONTRIBUTING.md](../CONTRIBUTING.md#3-风险分类与快速通道)。

## 适用范围与职责

适用于 `Shell/` 的应用组合根、模块发现与注册、启动/退出、全局异常、运行时依赖复制和主窗口生命周期。调查入口见 [变更影响地图](../docs/development/change-impact-map.md)，验证入口见 [构建与测试地图](../docs/development/build-and-test-map.md)。

## 修改前调查

- 定位模块扫描/注册、配置加载、服务容器、启动顺序、全局异常、窗口关闭、后台任务和最终释放。
- 检查 Application、Core、Tools、Hardware、客户模块、Native/厂商 DLL 与 `OutputExe` 的装配和复制关系。
- 区分项目引用、文件引用、反射加载和输出目录发现；任何一种都不能单独代表完整运行时依赖。

## 升级条件

- 模块身份、加载顺序、配置、公共服务或启动/退出契约变化至少 R2。
- 发布配置、权限、安全停止、设备初始化或运行时复制策略变化至少 R3。
- Shell 不得把 UI 可见性当作后端权限/互锁，也不得在未授权时连接或使能真实设备。

## 最低验证

- 先构建 Shell，再构建或运行受影响模块的直接测试；按影响覆盖模块发现、启动失败、部分加载、全局异常、取消和退出释放。
- 验证 Debug/Release、目标架构、依赖复制和缺失/错误版本行为；厂商依赖缺失时如实标记未验证。
- 交付说明模块、配置、输出、发布和回滚边界，不把编译成功表述为启动或发布验证。
