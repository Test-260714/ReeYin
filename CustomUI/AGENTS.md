# CustomUI 领域规则

本文件继承仓库根级 [AGENTS.md](../AGENTS.md)，只能收紧或细化根级门禁，不得放宽安全、验证、评审、授权或例外要求。规则冲突时执行更严格者并停止冲突范围内的修改，交由负责人裁决。

## 适用范围

适用于 `CustomUI/` 及其全部子目录中的项目、业务或客户组合 UI、页面、ViewModel、局部行为/转换器、Prism 导航、基于共享资源的扩展、测试和文档。当前全局共享控件、主题、样式和资源字典的权威所有者是 `Core/ReeYin_V.UI`；更深层 `AGENTS.md` 可以继续收紧本文件。

## Prism 导航、Binding 与线程

- 导航名、区域名、参数键和页面身份是运行时契约。模块入口注册导航类型一次；`OnNavigatedTo` 校验参数存在性和类型，`OnNavigatedFrom`/关闭路径释放页面拥有的订阅与短期资源。重复导航、返回、重新打开和失败导航不得累积实例副作用或丢失状态。
- View 只负责呈现和纯交互，ViewModel 负责状态、命令和可测试逻辑。新增或修改 Binding 必须核对真实属性、`DataContext`、Mode、`UpdateSourceTrigger`、验证和 `CanExecute`，并在运行时检查 Binding/命令错误；禁止以 XAML 编译代替诊断。
- `DependencyProperty`、绑定源和 `ObservableCollection` 的 UI 变更必须在 Dispatcher/UI 线程执行。耗时算法、文件/数据库、网络和设备等待不能阻塞 UI 线程；后台任务必须可取消、可观察异常并在页面离开或所有者释放时终止。
- 事件、命令、计时器、行为和 Prism 订阅必须有明确所有者。保存原委托或订阅令牌并幂等解绑；长生命周期发布者不得隐式保活短生命周期 View/ViewModel，释放后不得继续更新控件。

## 共享资源、缩放与可访问性

- 必须复用 `Core/ReeYin_V.UI` 的共享控件、样式、主题刷、字体和转换器；`CustomUI` 只在项目、业务或客户组合层扩展。主题颜色使用 `DynamicResource`；局部样式仅在 `BasedOn` 共享样式的小范围扩展或已有页面家族兼容变体中使用，不得复制整套模板、硬编码主题颜色或重定义全局资源键形成第二权威来源。
- 当前共享资源键、默认控件样式和模板部件名由 `Core/ReeYin_V.UI` 拥有，导航标识由其声明模块拥有。目录名称不表示共享 UI 已迁移到 `CustomUI`；修改组合 UI 时必须验证资源缺失、覆盖顺序及亮/暗/定制主题，并保持现有共享资源 URI 和加载路径。
- 未来将共享 UI 所有权迁移到 `CustomUI` 必须单独完成至少 R2 的设计、全部可定位消费者清单、资源/程序集身份迁移、兼容窗口、构建与运行时加载验证和回滚；迁移未完成前不得改变 `Core/ReeYin_V.UI` 的权威所有权。共享 UI 本体变更同时适用 [Core 领域规则](../Core/AGENTS.md)。
- 布局必须在 Windows 常用缩放/DPI、窗口尺寸、长文本和多语言下保持可见、可操作且不遮挡关键状态；避免依赖固定像素、单一字体度量或颜色作为唯一信息载体。
- 交互控件必须具有可理解名称/标签、合理 Tab 顺序、键盘操作、可见焦点、足够对比度和状态反馈。错误、运行、禁用、取消和权限拒绝要可诊断；动画/高频刷新不得妨碍操作或掩盖报警。

## 硬件与数据边界

- UI 和 code-behind 禁止创建、连接、初始化、使能、复位、驱动或拥有相机、控制卡、PLC、IO、激光、真空、温控等硬件资源，也不得直接持有数据库事务或长期 Native 句柄。UI 只通过有明确所有者、安全校验、超时/取消/停止语义的命令或服务发出意图并展示状态。
- UI 禁用、确认框、权限可见性和提示不能替代服务端/后端权限、限位、互锁、报警、急停或数据完整性校验。命令失败、取消、超时、停止和权限拒绝不得转换为成功状态。
- 真机动作仍需限定对象、动作、执行人、窗口、检查及停止/回滚/恢复的真实设备操作授权；生产数据变更必须取得相同限定范围的生产数据变更授权。相关风险为 R4 时还需具体执行批准，AI 不得连接/操作真机、修改生产数据或执行 R4 操作。

## 修改该目录必须完成

以下清单按实际影响和 R0-R4 执行；不适用项必须记录原因，不能虚构为通过。纯 R0 文档变更执行 L0 差异、内容、路径和链接检查；R3/R4 的领域/安全评审、变更负责人批准、停止/回滚、风险披露及 R4 具体执行批准仍按根级强门禁执行。

- [ ] 写明页面/控件消费者、导航与 Binding 契约、线程和资源所有者、成功标准、非目标及最高风险等级。
- [ ] 检查导航注册、参数校验、重复导航/返回/关闭、事件解绑、任务取消和幂等释放。
- [ ] 涉及代码、XAML 或资源行为时，构建修改项目及可定位消费者，运行相关测试，并检查运行时 Binding/命令错误和异常/取消路径。
- [ ] 仅在 UI 行为或视觉相关变更时，人工验证页面打开、关键交互、键盘/焦点、可访问名称、长文本、多语言、窗口尺寸、DPI/缩放、亮/暗/定制主题及关闭恢复。
- [ ] UI 行为或视觉人工证据记录步骤、环境、预期/实际、执行人/时间、适用的脱敏截图/日志和未覆盖分支；XAML 编译结果不替代视觉或交互证据。
- [ ] 确认 UI 不拥有硬件/数据库/Native 长期资源，真机或生产数据动作只经受控服务并保持独立授权和 R4 边界。
- [ ] 交付公共资源、导航、Binding、用户行为和设备/数据边界兼容结论，以及回滚步骤、失败/未验证项、残余风险和所需人类评审/批准。
