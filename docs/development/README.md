# ReeYin-V 开发规范索引

本目录把仓库根级 [强制规则](../../AGENTS.md) 和 [统一贡献流程](../../CONTRIBUTING.md) 展开为可执行的专题规范。所有贡献者（人类、AI 和自动化工具）均必须先遵守根级门禁，再读取与目标文件最接近的 `AGENTS.md` 和相关专题；子目录规则只能增加约束，不能降低根级要求。

## Quick Mandatory Checklist（强制快速清单）

开始修改前、实现中和交付前必须逐项执行：

- [ ] 写明当前问题或需求、成功标准、非目标、影响对象和已知约束；Bug 先保留可复现证据，不能复现时记录已尝试条件与证据缺口。
- [ ] 检查工作区状态，识别已有修改、未跟踪文件和生成物；保护不属于当前任务的用户改动。
- [ ] 阅读根级规则、目标目录的就近规则、贡献流程和相关专题；确认没有互相冲突的指令。
- [ ] 从入口调查到副作用：调用方、被调用方、公共消费者、配置/序列化/Recipe/Output/缓存/数据库、线程、异常与资源释放路径。
- [ ] 按“取最高等级”原则判定 R0–R4；命中设计触发器时，先形成包含兼容、失败/恢复、安全、验证和回滚的设计并取得所需评审。
- [ ] 只修改满足成功标准的最小文件与行为；同步处理输入校验、失败、取消、超时、停止、恢复、并发和释放路径。
- [ ] 按风险等级从差异/静态检查开始，逐层执行相关测试、消费者构建和场景验证；真实设备或生产数据操作必须另行获得明确授权。
- [ ] 使用最终修改后的新鲜证据自审并交付；如实列出未执行或失败的验证、兼容性、回滚方式、残余风险和所需评审/批准。

## Risk Classification（风险分类）

一个变更命中多个类别时采用最高等级；无法确定时按较高等级处理。实施中风险升高时必须暂停，重新确定设计、验证、评审和授权。

| 等级 | 典型范围 | 最低门禁概览 |
| --- | --- | --- |
| R0 | 文档、注释且不改变运行行为 | 差异、内容、路径和链接检查 |
| R1 | 局部 UI、内部逻辑、非公共重构 | R0 + 修改项目构建 + 相关测试 |
| R2 | 公共 API、MVVM 生命周期、Recipe、Output、序列化、缓存、数据库 | R1 + 消费者构建 + 兼容性与场景测试；涉及公共契约或兼容边界时由熟悉该领域的人评审 |
| R3 | 硬件运动、报警、权限、安全、迁移、发布配置 | R2 + 失败/恢复路径 + 回滚方案 + 对应领域评审 |
| R4 | 对设备、人身或生产数据存在不可逆风险 | R3 + 真实环境验证方案 + 明确负责人批准；AI 不得执行 R4 操作 |

风险等级只规定最低门禁。实际影响更广、失败代价更高或证据不足时，必须增加消费者、兼容、恢复或人工场景验证，不能以下调等级替代验证。

## Rule Exceptions（规则例外）

例外只能处理确实无法执行的门禁，不能用来降低风险等级、隐藏失败或把“未验证”改写为“通过”。申请时必须记录：未执行门禁、原因、影响范围、替代验证、残余风险、建议补验环境、批准人和有效期限。

- 安全、人身、设备、凭据、权限、生产数据底线不可被例外削弱；禁止绕过急停、限位、互锁、超时、停止、TLS、证书校验、审计和输入校验。
- R3/R4 禁止豁免安全评审、回滚/停止方案、负责人批准和风险披露；R4 操作不得交给 AI 执行。
- 紧急情况下只能缩短或延期非安全验证，并必须记录原因、替代验证、批准人、批准有效期限和补验时限；逾期必须撤销例外或停止交付/运行。
- 缺少硬件、许可证、依赖或权限只能形成“未验证”结论。AI 可以记录并上报例外请求，但不得批准例外或代替领域负责人接受风险。
