# 安全与信息安全

## 必须做

- [ ] 识别变更是否涉及真实设备、运动、报警、互锁、权限、生产数据、凭据、网络、遥测、外部进程或第三方依赖，并按最高风险定级。
- [ ] 在任何真实设备连接、初始化、使能、复位、采集、写 IO 或运动前取得针对设备、操作、人员和窗口的真实设备操作授权；风险分类为 R4 时升级为 R4 执行批准。
- [ ] 在任何生产数据修改、迁移、删除或恢复前取得针对数据目标、操作、人员和窗口的生产数据变更授权；风险分类为 R4 时升级为 R4 执行批准。
- [ ] 对运动逐项核对轴、单位、方向、速度/加速度、软硬限位、互锁、停止和急停；未经确认禁止发出动作。
- [ ] 保留报警、超时、取消、停止、恢复和失败语义；禁止吞掉、降级或转换为成功后继续危险流程。
- [ ] 对源码、日志、截图、测试数据和交付报告进行脱敏，禁止写入真实凭据、个人信息和生产敏感数据。
- [ ] 对新增网络访问、遥测、上传、外部进程、脚本和依赖来源进行显式风险、权限、许可证与回滚审查。
- [ ] R3 取得人类领域评审通过和变更负责人批准后才能交付或进入受控验证；R4 再取得覆盖具体设备或数据目标、操作、执行人、窗口、检查及停止/回滚/恢复的执行批准，AI 不得执行。
- [ ] 交付时记录安全前提、停止/回滚方案、未验证项、残余风险、批准人和适用限制。

本规范细化根级 [安全底线](../../AGENTS.md) 与 [统一贡献流程](../../CONTRIBUTING.md)。验证层级和证据边界见 [测试与验证](testing-and-verification.md)，审批和例外记录见 [评审与交付](review-and-delivery.md)，AI 的权限限制见 [AI 开发规范](ai-development.md)。

## 风险与责任

硬件运动、报警、权限、安全、迁移和发布配置通常至少为 R3；对设备、人身或生产数据存在不可逆风险时为 R4。一个变更同时命中多个类别时取最高等级，不确定时按较高等级处理。

- R2 涉及公共契约或兼容边界时，必须由熟悉该领域的人类作出领域评审结论。
- R3 必须覆盖失败/恢复路径、停止与回滚方案，由硬件、安全、数据或发布等对应领域的人类评审通过，并由明确的变更负责人批准后才能交付或进入受控验证。
- R4 还必须有真实环境验证方案，并取得覆盖具体设备或数据目标、操作、执行人、窗口、检查及停止/回滚/恢复的 R4 执行批准。批准不得扩大到其他对象、动作或窗口，AI 不得执行任何设备、数据或其他对象的 R4 操作。
- AI 不得连接、使能、驱动或操作真实设备，也不得代替人类确认设备状态、接受人身/生产风险或批准例外。
- 安全责任人、设备责任人、数据责任人和发布责任人按实际影响分别负责领域判断；作者不得以自审代替所需的独立领域评审。

领域评审是技术、安全或兼容性的 sign-off 结论；变更批准是允许交付、合入或进入下一阶段。基础受控操作授权按对象分为真实设备操作授权、生产数据变更授权，以及任务明确要求的其他高风险对象授权；每项授权必须限定具体设备或数据目标、操作、执行人、窗口、检查及停止/回滚/恢复。风险分类为 R4 时，相关受控操作授权升级为 R4 执行批准并满足 R4 额外门禁。例外批准只允许在有效期内延期非安全验证，不能替代领域评审、变更批准、受控操作授权或 R4 执行批准。

## 真实设备操作授权与隔离

未经明确的真实设备操作授权，禁止连接、初始化、使能、复位、改变配置、触发采集、写 IO、运动或以其他方式改变真实设备状态。授权必须限定设备类别/身份、动作范围、环境、执行人、时间窗口和停止条件；风险分类为 R4 时，该授权必须升级为 R4 执行批准。真实设备操作授权不能由领域评审、变更批准、例外批准、“代码已完成”或“测试通过”推定。

真实设备操作前必须确认：

- 设备身份、固件/驱动/SDK 和目标配置与批准方案一致；
- 人员、工件、夹具和周边区域处于安全状态，危险区域已隔离；
- 权限、物理急停、软硬限位、互锁、报警和安全回路可用且未被旁路；
- 单位、坐标系、轴映射、方向、速度、加速度、行程和起始状态已由负责人复核；
- 通信/动作超时、取消、停止、断线、掉电、异常返回和恢复路径已定义；
- 日志、采集和截图已按数据分类脱敏，且不会暴露设备敏感标识或生产信息；
- 安全停止、回滚和恢复的执行人、触发条件与联系方式已明确。

模拟器、桩、回放和离线数据必须与真实设备环境隔离。配置、连接字符串、设备地址和运行入口不得让模拟验证意外落到真机；真实设备未授权或不可用时必须记录“真实设备未验证”。

## 运动与执行器安全

运动控制、机械执行器、激光、真空、温控及其他可能产生物理副作用的能力必须采用防御性边界：

- **轴与坐标**：核对逻辑轴、物理轴、坐标系、原点和轴序；禁止用数组位置、默认轴或旧配置猜测映射。
- **单位与方向**：位置、速度、加速度、时间、角度、压力和温度必须有明确单位；方向、正负号、绝对/相对模式必须在命令边界校验。
- **速度与范围**：使用经批准的速度/加速度和行程；首次或恢复验证使用负责人批准的保守参数，禁止通过提高限值规避超时。
- **限位与互锁**：软限位、硬限位、门禁、夹具、真空、温度等互锁必须在动作前和动作中保持有效；UI 禁用或提示不能替代后端校验。
- **停止与急停**：普通停止、安全停止和急停的语义、优先级、幂等性及作用范围必须明确。任何失败、取消、超时或通信中断不得留下未观察的运动。
- **恢复**：急停、限位、掉电、断线或异常后的恢复必须先重新确认状态和授权；禁止自动清除报警、自动复位或自动重发有副作用命令。
- **重试**：运动、写 IO 和其他非幂等动作默认不得自动重试。确需重试时必须由设计说明幂等条件、次数、退避、取消和停止路径并取得领域评审。

禁止为调试或验证而绕过急停、限位、互锁、权限或安全回路。任何这类绕过请求都必须停止相关工作并交由负责人裁决。

## 报警、异常与停止语义

- 报警、设备错误、权限拒绝、超时和数据完整性错误不得被空 `catch`、仅写日志、默认值或成功状态吞掉。
- 报警必须保留来源、稳定标识、严重度、发生时间和可执行上下文；日志不得包含敏感数据。安全报警不能降级到不可见级别来减少噪声。
- 无法安全继续时必须阻止后续动作，触发拥有该操作的停止/清理路径，并把失败传播到调用方和可见状态。
- 超时必须有明确时限、作用范围和超时后的安全状态。取消与停止不得被当作成功，也不得因异常处理而跳过资源释放。
- 恢复必须验证前置状态、资源所有权和重复执行安全性；禁止在状态未知时自动恢复运行。
- 清理失败不得覆盖主要故障；需要同时报告时，保留两者的诊断信息并维持安全停止状态。

## 生产数据、凭据与隐私

禁止在源码、配置样例、日志、截图、测试数据、命令输出和交付报告中写入或泄露真实密码、令牌、密钥、证书私钥、完整连接字符串、个人信息、客户数据、生产配方、设备标识或其他生产敏感数据。

- 使用明确虚构或脱敏的测试数据；脱敏必须避免通过路径、时间、组合字段或截图背景重新识别。
- 凭据必须通过批准的安全配置或秘密管理机制提供，不得硬编码、提交、复制到脚本参数或输出到日志。
- 生产数据的读取、复制、修改、迁移、删除和恢复都需要生产数据变更授权、最小权限、备份/恢复方案和审计记录；授权必须限定数据目标、操作、执行人、窗口、完整性检查及停止/回滚/恢复，风险分类为 R4 时升级为 R4 执行批准。
- 禁止以生产数据作为试迁移或调试夹具。测试副本必须经批准并脱敏，使用后按数据保留规则处理。
- 迁移和发布配置属于 R3；若可能造成不可逆数据损失或生产风险则为 R4。失败、事务回滚、版本识别和恢复必须在非生产环境先验证。
- 发现疑似凭据或敏感数据泄露时，停止继续传播，保留最小必要证据，并通知具备权限的负责人按事件流程处置；AI 不得自行轮换生产凭据或修改生产系统。

## 网络、遥测与外部进程

新增或改变网络请求、上传、遥测、自动更新、远程调用、外部进程、命令行或脚本执行前必须显式评审：

- 业务必要性、目标主机/进程、数据类别、协议、身份认证、TLS 与证书校验；
- 最小权限、输入校验、命令/参数边界、超时、取消、重试和资源限制；
- 离线、代理、DNS/证书失败、部分响应、进程挂起/崩溃及停止/清理路径；
- 是否收集、上传或持久化数据，用户/负责人知情与授权、保留期限和脱敏；
- 日志、审计、可禁用方式、回滚和对生产/设备实时性的影响。

禁止关闭 TLS、证书校验、权限、审计或输入校验来使功能通过。外部进程不得使用不受信任输入拼接可执行文件、脚本或参数；进程必须有明确来源、工作目录、超时、退出码处理和终止/清理责任。

## 依赖来源与供应链

新增、替换或升级包、SDK、Native 库、二进制、工具和脚本必须作为独立范围审查：

- 来源、维护者、精确版本、完整性校验与获取方式；
- 许可证、分发限制、已知安全问题和支持生命周期；
- API/ABI、目标框架、操作系统/架构、厂商驱动和运行时复制兼容；
- 安装/构建是否产生网络访问、遥测、外部进程或机器级状态变化；
- 消费者构建、运行场景、发布复制、升级失败和回滚到旧版本的方式。

禁止使用来源不明的二进制、真实许可证密钥、个人机器绝对路径或未经审查的下载地址。缺少许可证、依赖或来源证据时只能形成“未验证”结论，不能用本地偶然可用替代审查。

## 不可豁免门禁

R3/R4 禁止豁免以下内容：

- 安全评审；
- 可执行的停止与回滚方案；
- R3/R4 所需领域评审通过和变更负责人批准；
- R4 针对具体设备或数据目标、操作、执行人、窗口、检查及停止/回滚/恢复的执行批准；
- 已知和残余风险披露。

人身、设备、急停、限位、互锁、权限、TLS、证书校验、审计、凭据和生产数据底线不得通过任何例外削弱。例外批准只可以在紧急情况下限期缩短或延期非安全验证，并按 [评审与交付](review-and-delivery.md) 记录原因、范围、替代验证、风险、例外批准人、有效期限和补验期限；它不能替代领域评审、变更批准、受控操作授权或 R4 执行批准。逾期必须撤销例外或停止交付/运行。AI 可以记录例外请求，但不得批准例外或接受风险。
