# 编码标准

## 必须做

- [ ] 编辑前阅读目标项目与至少一个相邻实现，确认 `.NET 8 + WPF + Prism` 的当前局部模式、项目设置和适用领域规则。
- [ ] 明确正常、失败、取消、超时、停止、恢复、并发与释放路径；为每个订阅、句柄、任务和缓存指定所有者。
- [ ] 保持公共 API、序列化、Recipe、Output、缓存键、数据库、模块加载和 UI 行为兼容；需要改变时先走设计和评审。
- [ ] 对边界输入做空值、范围、单位、状态和线程校验；不得把硬件、权限或数据错误转换为成功。
- [ ] 在最终修改后构建受影响项目，运行相关测试和场景检查，检查完整警告/错误输出并记录新鲜证据。
- [ ] 自审异常、日志、事件取消订阅、`IDisposable`、Binding、反射/序列化和警告抑制；确认没有敏感信息和无关格式化。

## 基本原则

优先遵循当前项目和相邻文件的局部风格，包括缩进、花括号、成员排序、命令/属性通知和 XAML 布局；不要为统一个人偏好批量格式化。局部风格与安全底线、公共契约或明确生命周期要求冲突时，强规则优先：记录冲突，限定修正范围，并在需要时取得设计或领域评审。

实现必须让所有权、线程、失败语义和兼容边界可以从类型与调用关系中判断。禁止用反射、全局状态、静默异常或 UI 事件绕开正常模块契约。

## C#

- 新代码使用目标项目当前 C# 语言版本和 `net8.0-windows` 能力，不得仅因个人偏好改动全局语言版本、目标框架或项目设置。
- 类型和方法保持单一职责；提取新抽象前确认至少有明确边界或复用需求。禁止把业务逻辑放入 `Shell` 启动代码、WPF code-behind 或通用工具类来规避正确归属。
- 优先使用强类型接口、DTO、枚举和现有 Prism/DI 注册；禁止以 `dynamic`、字符串类型名或服务定位器隐藏可表达的编译期依赖。
- 参数和返回值应表达单位、范围和失败语义。涉及位置、速度、时间、图像尺寸或设备通道时，名称或类型必须避免单位歧义。
- 属性变更使用项目现有的 `BindableBase`/`SetProperty`/`RaisePropertyChanged` 模式；只有值实际变化时才发通知，派生属性的通知必须覆盖所有来源。
- 集合暴露与并发访问策略必须明确。不要把可变内部集合直接作为跨模块契约，也不要在枚举期间无同步修改集合。

## 命名与布局

- 公开类型、成员、命令和事件使用 PascalCase；参数和局部变量使用 camelCase；私有字段沿用目标项目现有前缀风格。
- 接口使用 `I` 前缀；异步方法以 `Async` 结尾，事件处理器和框架约定签名除外；布尔名称应表达肯定状态，避免含糊缩写。
- 命名必须表达领域语义，不得继续引入 `Serail` 等拼写错误。修正已有公开拼写前先评估序列化、Binding、反射和消费者兼容。
- 文件通常只承载一个主要类型，文件名与主要类型一致；局部辅助类型仅在不形成复用或契约时同文件放置。
- `using`、命名空间形式、成员顺序和表达式体沿用目标项目；禁止为了风格统一重排未修改代码。

## Nullable 与输入边界

- 遵循目标项目当前 nullable 设置。禁止为消除警告全局关闭 nullable，也禁止未经证明使用 null-forgiving 运算符 `!`。
- 新增或修改的边界必须真实表达可空性；项目未启用 nullable 注解时，仍必须在运行时校验外部输入、反序列化数据、DI/导航参数和硬件返回值。
- `as`、`FirstOrDefault`、字典查询、服务解析、导航参数和反序列化结果必须在解引用前检查。字典优先使用 `TryGetValue`，并验证实际类型。
- 只有框架能够保证且证据可定位时才使用 `!`；在相邻注释或设计中记录非显然保证。不得用默认对象掩盖缺失配置或失败状态。
- 公共参数的空值行为保持兼容；从允许 `null` 改为抛异常或反向改变都属于行为变更，必须检查消费者。

## Async 与线程

- 异步链保持“异步到底”。除事件处理器等框架签名外禁止 `async void`；返回 `Task`/`Task<T>` 并让调用方观察完成、异常和取消。
- 禁止无理由使用 `.Result`、`.Wait()` 或 `GetAwaiter().GetResult()` 同步阻塞异步工作。确有同步框架边界时，必须证明不会在 UI/受限同步上下文死锁，记录原因，并把阻塞限制在单一边界。
- 可取消操作必须接收并传递 `CancellationToken`；外部 IO、硬件、网络或可能无限等待的操作必须有明确超时和安全停止路径。取消不是成功，也不应默认记录为错误。
- WPF 控件、`ObservableCollection` 和绑定源的 UI 变更必须在 Dispatcher/UI 线程执行；耗时计算、阻塞 IO 和设备等待不得占用 UI 线程。
- 后台任务必须由可定位对象拥有并被等待、取消或观察；禁止 fire-and-forget 丢失异常。确需后台常驻任务时，定义启动、停止、异常上报和释放协议。
- 共享状态必须有明确同步策略。不要用 `Task.Run` 掩盖线程不安全的 SDK、数据库连接或集合；锁内禁止执行不可控外部调用或同步等待异步操作。
- 重试必须有上限、退避、可取消性和幂等性判断；运动、写 IO、数据库写入等有副作用操作不得默认重试。

## 异常处理

- 只在能够补充上下文、转换为契约结果、恢复或清理的边界捕获异常；否则让异常传播到明确的所有者。
- 禁止空 `catch`、仅记录后继续危险流程或把异常静默转换为 `true`/成功状态。无法继续时返回明确失败并触发停止/清理。
- 重新抛出使用 `throw;` 保留堆栈；包装异常时保留原异常为 `InnerException`，并增加不含敏感信息的操作上下文。
- 分别处理预期取消、超时、输入错误、资源不可用和不可恢复故障；不要用 `catch (Exception)` 代替可执行的状态设计。
- `finally` 只做必须执行且可控的清理。清理失败不得覆盖主要故障；需要同时报告时保留两者的诊断信息。
- 全局异常处理是最后诊断边界，不是业务恢复机制；不得依赖 `Shell` 全局捕获来掩盖未观察 Task、UI 线程或设备异常。

## 日志

- 使用仓库现有 NLog 和局部日志模式；记录操作、稳定标识、状态变化、耗时和可执行的失败上下文，避免只有“失败了”的无定位信息。
- 日志级别与影响匹配：可恢复诊断信息不冒充致命错误，安全/硬件失败不得降级为不可见的调试日志。
- 禁止记录凭据、令牌、密钥、个人信息、未脱敏客户/生产数据、完整连接字符串或设备敏感标识。对象序列化和异常 `Data` 也必须审查。
- 高频循环、图像帧和设备轮询日志必须限频或聚合，不能影响实时性、磁盘或暴露大量数据；禁止用日志代替状态通知和错误返回。
- 记录异常时避免同一边界重复打印；由最能说明操作语义的一层记录，其他层保留传播。

## 事件与 Prism

- 订阅必须有对应取消订阅或与发布者相同/更短且可证明的生命周期。长生命周期发布者订阅短生命周期对象时必须显式释放。
- 保存原委托、命名处理器或 Prism 订阅令牌用于取消；禁止用新的匿名 lambda 执行 `-=`，因为它不会移除原订阅。
- 重复初始化、反序列化、导航往返和重复打开不得累积事件、命令或 `TriggerModuleRun` 委托。注册前使用幂等守卫或先释放旧注册。
- 事件负载保持小而稳定，不携带 WPF 控件、厂商资源或所有权不明的可释放对象。发布方不得依赖订阅顺序或未声明线程。
- UI 更新订阅必须明确切换到 UI 线程；事件处理器异常必须被拥有该操作的边界观察，不能破坏事件总线或静默丢失。

## IDisposable 与资源释放

- 当前代码创建并保留 `IDisposable`/`IAsyncDisposable`、事件订阅、计时器、取消令牌源、Native 指针、图像缓存、数据库事务或硬件句柄，或者通过明确契约接管其所有权时，必须同时确定唯一所有者和释放时机。
- 对本地创建且保留所有权的局部资源优先使用 `using`/`await using`；跨方法资源由拥有类型实现幂等 `Dispose`/`DisposeAsync`，释放子资源、停止任务和取消订阅。
- 从 DI 容器解析、属性 getter 或方法参数取得的对象默认是借用，消费者不得调用 `Dispose`/`DisposeAsync`；普通返回值也不表示所有权转移，除非工厂或契约明确说明。
- 不拥有的借用资源不得擅自释放；所有权转移必须通过契约、工厂或命名明确表达。共享或引用计数对象按契约释放当前引用，禁止多个对象都假定对同一硬件句柄或底层资源拥有最终释放权。
- 释放路径必须覆盖初始化部分失败、取消、超时、重复关闭、模块卸载、方案切换和应用退出。释放后禁止继续发布状态或访问 UI。
- 只有直接拥有非托管资源时才引入终结器；使用标准 Dispose 模式并调用 `GC.SuppressFinalize(this)`。不要依靠 GC 及时释放设备或文件资源。
- `ModelParamBase`、动态页面和输出资源的基类调用顺序按 [模块开发与生命周期](module-development.md) 执行，不能用通用 Dispose 模板覆盖框架生命周期。

## XAML 与 WPF/Prism

- View 负责呈现和纯交互，ViewModel 负责状态、命令和可测试的 UI 逻辑；业务、数据库和硬件调用通过服务/Model 进入，不放入 code-behind。仅视觉行为或框架事件桥接可留在 code-behind。
- 共享控件、样式、主题资源和转换器的权威来源是 `Core/ReeYin_V.UI`。`CustomUI` 仅承载项目、业务或客户组合层的局部扩展，必须复用该共享资源，不得作为通用共享依赖或第二权威来源；将共享 UI 所有权或通用资源迁入 `CustomUI` 必须作为独立的至少 R2 设计，覆盖可定位消费者迁移、兼容窗口、运行时加载验证和回滚。新增局部样式时基于共享样式，禁止无理由硬编码颜色、字体和重复模板。
- Binding 必须使用真实属性名、正确 `DataContext`、模式和更新时机；新增或修改页面要检查运行时 Binding 错误、命令 CanExecute 和验证提示。
- Prism 导航参数必须校验存在性和类型；导航进入/离开时管理订阅与资源。ViewModel 构造函数不得启动依赖完整导航参数或有效节点 Serial 的副作用。
- UI 必须保持可操作：耗时命令显示运行/禁用状态，失败可诊断，取消/关闭走安全路径；不得用禁用按钮替代后端互锁和权限校验。
- 修改页面后至少检查页面打开、关键交互、键盘/焦点、不同缩放/DPI、主题和长文本；人工 UI 结果按验证规范记录，不能以 XAML 编译代替。

## 反射、序列化与动态加载

- 使用反射或属性收集器前确认成员可见性、可写性、继承、默认值和异常行为；缓存反射结果时，键和失效策略必须明确且线程安全。
- 反射不得访问其他模块内部实现来绕过依赖边界。类型名、程序集名、导航名和成员路径一旦持久化或由配置引用，就按公共契约管理。
- 序列化变更必须验证新数据往返、旧数据读取、缺失/额外字段、默认值和错误数据；不得把反序列化成功等同于语义有效。
- `[RecipeParam]` 不得与导致收集器跳过成员的 `[JsonIgnore]` 组合；`[OutputParam]` 标记可写成员。名称、路径、类型或刷新时机变化按 R2 兼容边界处理。
- 动态创建类型或加载程序集必须限定来源、允许范围、失败语义和卸载/释放路径；禁止从不受信任输入拼接类型或执行任意代码。

## 注释与文档

- 注释解释“为什么”、不变量、安全前提、单位、兼容原因或非显然生命周期；不逐行复述代码，也不保留失效实现和大段注释代码。
- 公共契约、反常行为、硬件前置条件和迁移窗口需要与消费者可见的文档；注释不能代替输入校验、类型或测试。
- 修改行为时同步更新直接相关注释和文档。无法立即处理的旁支问题应在交付中报告，不在源码中添加无负责人、无验收条件的模糊占位符。
- 注释、日志和示例使用脱敏数据；不得包含真实凭据、客户数据、生产路径或设备标识。

## 生成文件与第三方代码

目录级保护规则见 [`thirdparty/AGENTS.md`](../../thirdparty/AGENTS.md)、[`packages/AGENTS.md`](../../packages/AGENTS.md)、[`OutputExe/AGENTS.md`](../../OutputExe/AGENTS.md) 和 [`Resource/AGENTS.md`](../../Resource/AGENTS.md)。这些规则不授权手工修补生成物或依赖缓存。

- 识别设计器、代码生成器、资源编译器、数据库工具或构建产生的文件。默认修改生成源或模板，然后用受控命令再生成；禁止手工改生成物来制造短期一致。
- 只有任务明确包含生成物且评审/验证覆盖来源、工具版本和差异时才提交生成结果。不得顺带刷新无关生成文件或 `OutputExe` 产物。
- `thirdparty`/`packages` 中的源码和二进制不是本项目风格修正目标；修改、替换或升级必须单独审查来源、许可证、安全、ABI/API、架构和发布复制。
- 不得为生成或构建写入真实密钥、机器专属绝对路径或未审查下载地址。

## 警告策略

- 编译警告必须逐项阅读并判断是否由本次变更引入。新增警告在交付前修正；既有警告与本次结果分开记录，禁止隐藏或删除证据。
- 禁止全局关闭编译警告、nullable、分析器、安全检查或将警告级别整体降低来使构建成功；也不得无理由修改 `Directory.Build.props` 或项目级 `NoWarn`。
- 确需抑制确定的误报时，使用最小作用域、指定警告 ID，并在邻近位置写明不可由代码消除的原因和安全依据；抑制范围扩大时重新评审。
- 禁止用 `#pragma warning disable` 包围整个文件、复制未知抑制或把运行时错误改成警告后忽略。依赖/生成代码警告应从正确的引用或生成配置解决。
- 构建证据必须记录精确命令、执行时间、环境、退出码以及错误/失败/跳过计数；工具不提供的字段写“不适用”，不得猜测。
