# Semiconductor 领域规则

本文件继承根级 [AGENTS.md](../AGENTS.md)，只增加半导体领域模型、脚本和流程约束。风险定义见 [CONTRIBUTING.md](../CONTRIBUTING.md#3-风险分类与快速通道)。

## 适用范围与职责

适用于 `Semiconductor/` 的领域脚本、模型、测试工具和流程验证。目录当前包含 Roslyn scripting 项目并引用 `Application/ReeYin_V.Share`；实际消费者仍须从 [变更影响地图](../docs/development/change-impact-map.md) 和代码核对。

## 修改前调查

- 定位脚本入口、允许的 API/引用、领域参数、状态转换、配置/数据来源、异常传播和取消/停止路径。
- 检查输入是否来自方案、客户数据、设备或外部脚本；所有不可信脚本和数据必须验证边界并限制权限。
- 核对 Application/CustomizedDemand 消费者和脚本编译、缓存、加载、卸载行为。

## 升级条件

- 领域数据、脚本契约、流程状态、持久化或跨模块 API 变化至少 R2。
- 工艺、安全、报警、权限或设备动作语义变化至少 R3；不可逆真实操作为 R4。

## 最低验证

- 构建修改项目，验证脚本编译、允许/拒绝的引用、非法输入、异常、取消和重复执行。
- 数据/流程变化覆盖旧数据、边界状态、失败恢复和消费者兼容；设备相关只做授权范围内的模拟验证。
- 交付区分脚本/离线/模拟与真实环境证据。
