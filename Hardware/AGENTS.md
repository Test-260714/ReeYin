## 受控操作与模拟隔离

- 阅读、编译、静态分析、桩和离线模拟不授权真实设备操作。连接、搜索后自动连接、初始化、使能、复位、改变配置、触发采集、写 IO、运动或其他真实状态改变前，必须取得限定设备身份/类别、动作、环境、执行人、窗口、检查及停止/回滚/恢复的真实设备操作授权。
- 对设备、人身或生产数据存在不可逆风险时按 R4 处理：先满足 R3 的人类领域评审通过、变更负责人批准、失败/恢复和停止/回滚门禁，再取得覆盖具体对象与动作的 R4 执行批准。AI 不得连接或操作真实设备，也不得执行任何 R4 操作。
- 模拟器、桩、回放和离线数据必须与真机在配置、连接字符串/地址、运行入口、凭据和输出位置上明确隔离，并默认阻止意外落到真机。证据必须标记“模拟验证”“离线验证”或“真实设备验证”；前两者和编译结果不能证明厂商 SDK 时序、物理互锁或真实设备行为。
- 若硬件路径读取或写入生产数据，还必须取得独立的生产数据变更授权；相关风险为 R4 时升级为 R4 执行批准。设备授权、数据授权、领域评审、变更批准和例外批准不能互相替代。

## 抽象、失败与安全停止

- 公共硬件契约必须表达设备状态、单位、范围、超时、取消、停止、错误和所有权；厂商 SDK 类型、句柄、返回码和线程限制封装在厂商适配层。业务、Tools 和 UI 只消费稳定抽象，不直接创建或长期保存厂商对象。
- 任何 IO、采集、连接、等待或动作都必须有明确超时和取消/停止路径。重试必须有限次、退避、可取消并证明幂等；运动、写 IO、复位和其他非幂等动作默认不得自动重试或断线后重放。
- 普通停止、安全停止和急停的语义、优先级、范围与幂等性必须明确。动作前和动作中都要保持权限、报警、软硬限位、门禁及其他互锁有效；UI 禁用、提示或确认框不能替代后端互锁。失败、取消、超时、断线或通信异常必须收敛到已定义的安全状态。
- 异常、厂商错误码、权限拒绝、超时、报警、取消和停止不得转换为成功或默认值后继续。无法安全继续时必须阻止后续副作用、执行拥有者的停止/清理并向调用方传播可诊断的失败；清理故障不得覆盖主要故障。

## 初始化、事件、释放与重连

- 初始化和关闭必须幂等并覆盖重复调用、并发调用和部分初始化失败。只有连接、配置、状态订阅和必要安全检查均已建立后才能发布就绪；失败后必须释放本次取得的局部资源，使重试前状态明确。
- 设备句柄、采集流、运动会话、后台轮询、计时器、取消令牌源和事件订阅必须有唯一所有者。保存原委托或订阅令牌；关闭、模块卸载、方案切换和应用退出时先停止动作/任务，再解绑并释放自有资源。借用的 DI/SDK 资源按契约处理，禁止重复释放。
- 断线后状态必须进入可观察的未就绪/未知状态并阻止新动作。重连应有限时、可取消并重新校验设备身份、配置、报警、限位、互锁和起始状态；禁止自动清除报警、自动复位、自动恢复运动或重发有副作用命令。
- 新增或更改厂商依赖必须单独审查来源、版本、许可证、安全、目标架构、驱动/固件、API/ABI、运行时复制和回滚版本；不得嵌入真实许可证或机器专属路径。

## 修改该目录必须完成

以下清单按实际影响和 R0-R4 执行；不适用项必须记录原因，不能虚构为通过。纯 R0 文档变更执行 L0 差异、内容、路径和链接检查；R3/R4 的领域/安全评审、变更负责人批准、停止/回滚、风险披露及 R4 具体执行批准仍按根级强门禁执行。

- [ ] 写明设备类型/身份类别、动作、单位/坐标、状态机、资源所有者、成功标准、非目标和最高风险等级。
- [ ] 保持厂商实现封装在稳定抽象后，检查消费者未直接依赖 SDK 类型、句柄或隐式线程约束。
- [ ] 覆盖离线、异常返回、取消、超时、停止、报警、断线、重连、重复初始化、部分失败和幂等释放。
- [ ] 对运动/执行器核对轴映射、坐标、方向、单位、速度/加速度、行程、软硬限位、互锁、停止和急停；未经受控操作授权不得进入真机步骤。
- [ ] 先构建最小受影响项目和消费者并运行相关测试，再用桩/模拟器验证失败与恢复；不得扩大模拟或编译证据的结论。
- [ ] 仅在验收需要真实设备结论或对应风险要求真机场景时，由获授权的人类执行真机人工验证，并记录步骤、环境、预期/实际、执行人/时间、脱敏日志/测量、具体授权和停止/回滚/恢复；R4 另关联具体执行批准。缺少设备、权限或授权时必须记录“真实设备未验证”及影响，不能声称真机验证成功。
- [ ] 交付时记录失败/未验证项、设备行为兼容性、真实设备未验证范围、残余风险、人类领域评审与变更批准；安全门禁和 R4 要求不可由例外削弱。
